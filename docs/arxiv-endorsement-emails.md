## Sending tips

- Send all three the same morning (parallel, not serial).
- Keep the subject line; some academics filter on it.
- Don't follow up before day 5. After day 5, a one-line "checking in" reply to
  the original thread is fine.
- If someone replies asking for clarification, send the link to the GitHub
  repo and the deployed product — those are more persuasive than the paper PDF
  for a first read.
